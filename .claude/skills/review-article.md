# 記事レビュー

指定された記事ファイル（引数またはカレントディレクトリの最新記事）をCLAUDE.mdの記事設計ガイドラインに基づいてレビューする。

## 手順

1. 引数でファイルパスが指定されていればそのファイルを、なければ `src/content/blog/` 内の最新の記事を読む。
2. CLAUDE.md の「記事設計ガイドライン」と「公開前チェックリスト」を参照する。
3. 以下の観点でレビューし、結果を報告する。

## レビュー観点

### frontmatter チェック

- title, description, date, categories, tags, permalink, published が揃っているか
- title にキーワードが含まれているか
- description が120文字程度で記事を要約しているか

### 構成チェック

- h2/h3 の階層構造が正しいか（h2 を飛ばして h3 にしていないか）
- キーワードが見出しの先頭寄りに配置されているか
- 目次を見ただけで記事の全容が理解できるか

### コンテンツ品質チェック

- 一次情報（体験、検証結果、プロセス）が含まれているか
- 書き手の主観的な意見・感想が盛り込まれているか
- AIが数秒で回答できる一般情報の再構成に留まっていないか

### リンクチェック

- 関連記事への内部リンクが設置されているか
- アンカーテキストが「こちら」ではなく具体的か
- 既存の関連記事がある場合、リンク候補を提案する

## 出力フォーマット

```
## レビュー結果: {記事タイトル}

### OK
- ...

### 改善提案
- ...

### 内部リンク候補
- ...
```
